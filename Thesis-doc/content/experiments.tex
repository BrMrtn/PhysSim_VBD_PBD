This chapter reports measurements along the four evaluation axes introduced in
\autoref{sec:EvalAxes}: efficiency, robustness, physical realism, and
flexibility. Unless stated otherwise, every experiment uses the shared
scaffolding and fairness controls of \autoref{sec:Fairness}: matched gravity,

Chebyshev acceleration disabled in VBD for the core comparison.

\subsection{Influence of substep and iteration count on error and compute time}
\label{ssec:ErrorVsMs}

For each solver I sweep over a range of substep and iteration configurations and
plot the Pareto front of the resulting runs --- the configurations that are not
dominated on both axes --- using average ms/frame against the average
root-mean-squared position error measured from a Newton reference. XPBD is swept
over substep counts $n_{\mathrm{sub}} \in \{1, 2, 4, 8, 16, 32\}$ and VBD over
iteration counts $n_{\mathrm{it}} \in \{1, 2, 4, 8, 16, 32\}$. Each diagram in
\autoref{fig:ErrorVsMs} uses a Newton reference computed with a different number
of substeps.

The test problem is a chain of 10 nodes with one end pinned and a weight at the
free end. Both ends start at the same height and only gravity acts on the chain.

A single VBD iteration approximates the Newton solution more closely than a
single XPBD iteration, but it is also more expensive. Against the four-substep
Newton reference, VBD run with the same substep count as the reference converges
very well. As the reference is given more substeps, however, iterations lose
their value: with a small enough time step the problem becomes easy enough for
both solvers to converge in one or two iterations, at which point XPBD takes
over thanks to its cheaper per-iteration cost.

\subsection{Self-collision: cloth folded onto itself}
\label{ssec:SelfCollisionDrop}

This scene drops a cloth onto the ground so that it folds onto itself. Both
solvers can simulate cloth, soft bodies, and rigid bodies under many constraints
and collisions, but physically accurate cloth requires handling stiff
constraints: woven fabrics, for instance, only stretch by about 5--10\%.

My cloth model attaches 12 springs with three different stiffness values to each
interior node. As the stiffness-ratio experiment above shows, this is a serious
disadvantage for VBD. The result is very unstable behaviour: even when the
simulation does not blow up, the cloth overshoots most configurations, with
initial vertex noise on the scale of $0.001$ producing large movements. This is
caused by the extra energy that the under-converged solve injects into the
system.

The instability can be tamed by giving the vertices a large enough mass ---
practical experiments suggest it needs to be 2--3 orders of magnitude smaller
than the maximum stiffness. This is exactly why the original VBD
paper~\cite{Chen2024} uses a different cloth model, and it shows that VBD
sometimes has to be hand-tuned to stay stable; AVBD~\cite{Giles2025} improves
substantially on this axis. \autoref{fig:SelfCollisionW1} and
\autoref{fig:SelfCollisionW1e4} show the drop at a total cloth mass of 1 and of
10000 respectively; in each, the top row is XPBD and the bottom row is VBD.

\subsection{Harmonic oscillator}
\label{ssec:Harmonic}

The harmonic oscillator is implemented as a single distance constraint between a
pinned top node and a free bottom node, and I measure its amplitude over time.
To compare against Fig.~2 of the XPBD paper~\cite{10.1145/2994258.2994272} I use
the same parameters, only with the direction reversed: rest length 1, compliance
$\alpha = 10^{-3}$, rest position $x = -1$, initial position $x_0 = -0.5$, and
mass $m = 1$.

Regardless of $dt$, both solvers converge in a single iteration and follow the
same curve. \autoref{fig:Oscillator} also shows the analytic solution
$x(t) = x_0\cos(\omega t)$ with $\omega = \sqrt{k/m} = 31.62~\mathrm{s}^{-1}$ as
dashed lines. As in the XPBD example, all of the visible damping is a
side-effect of the implicit discretisation of the equations of motion.

\subsection{Energy tracking}
\label{ssec:EnergyTracking}

Here I plot the total mechanical energy over time for an undamped $16\times 16$
cloth released from 20 degrees, in order to identify systematic energy gain
(instability) or loss (numerical damping) for each solver. I use VBD with 64
iterations as the reference solution, since it is very close to the Newton
solution with 64 substeps.

Over a longer time frame, \autoref{fig:EnergyLong} shows that both solvers lose
energy along the same exponential decay curve. The oscillations around that
curve are purely a convergence artefact: the solver over- and undershoots
because of its gradient estimate.

\subsection{Ease of adding a new functionality}
\label{ssec:NewFunctionality}

To compare how easily each solver can be extended, I had Claude Code implement
dihedral bending constraints in both. Both methods already include this
constraint in their original papers, so I follow that route. The existing
bending constraint is a distance constraint acting as a skip-spring between two
non-adjacent nodes; the task was to implement the dihedral version and make
bending toggleable between the dihedral and distance formulations.
\autoref{tab:Extension} summarises the effort.

The two implementations share a new 30-LOC file that computes the gradients from
the four hinge node positions, and the in-solver addition that solves the new
constraint is about 28 LOC and essentially identical in both. XPBD was easier to
extend: the new constraint resembles the existing distance constraint (four
particles instead of two), introduces no new concept, and plugs in with a single
extra \texttt{SolveDihedralConstraints} call without changing any existing code.
Each hinge is visited once per iteration. VBD needed more setup code because it
is vertex-based, so each hinge has to be stored four times --- once per
incident vertex --- and its data type carries an extra parameter naming which of
the four vertices is currently being processed. The hinge Hessian block is
therefore recomputed four times per iteration. Mathematically, XPBD needs only
the first derivative of the constraint, whereas VBD needs the second derivative,
which is harder to derive and implement.

In general, for scalar constraints XPBD is the easier solver to extend. For
tensor-valued energies (FEM, Neo-Hookean, StVK), it is XPBD that has to do the
extra work.

\subsection{Coverage of different physical domains}
\label{ssec:DomainCoverage}

PBD has been around for almost 20 years and XPBD for about 10, and both have
become industry standards for in-game physics. They have been implemented across
many domains --- cloth, soft bodies, rigid bodies, fluids, hair --- with a wide
range of constraint types and forces. XPBD copes with a large variety of
parameters and value imbalances, which makes it a very flexible solver.

VBD, by contrast, is a very new method, published in 2024, and is not yet widely
used; its original paper applies it mostly to elastic body dynamics. As the
self-collision experiment showed, it is not well suited to very hard (effectively
infinite-stiffness) constraints --- an issue that AVBD addresses by bringing
rigid bodies into the method.
