\subsection{Self-collision: cloth folded onto itself}
\label{ssec:SelfCollisionDrop}
\begin{itemize}
    \item A cloth dropped on the ground, folding onto itself.
    \item Both solvers can be used to simulate cloth, soft-bodies, rigid-bodies; a lot of constraints, and of course collisions.
    \item For a system to simulate physically accurate cloth, the handling of stiff constraints is needed, as some materials, like Woven fabrics only stretch up to around 5-10\%.
    \item The cloth model used by me requires a central node to have 12 springs attatched with 3 different stiffness values.
    \item This, seen from the stiffness ratio experiment above creates a huge disadvantage for VBD.
    \item This results in a very unstable behaviour, where even if the simulation doesn't blow up, the cloth overshoots most problems (initial noise to the vertices on the scale of 0.001 results in big movements).
    \item This is because of the extra energy in the system
    \item This can be handled by giving vertices a big enough mass (needs to be 2-3 orders of magnitude smaller than the maximum stiffness based on practical experiments).
    \item This is the exact reason why the original VBD paper~\cite{Chen2024} uses a different cloth model.
    \item This shows that for some situations VBD needs to be fine-tuned with parameters to be stable - AVBD~\cite{Giles2025} improves on this axis largely.
    \item The top row shows the cloth with a total mass of 1, the bottom row shows the same cloth with a total mass of 10000.
\end{itemize}

\subsection{Harmonic oscillator}
\label{ssec:Harmonic}
\begin{itemize}
    \item A harmonic oscillator implemented as a single distance constraint between the pinned top and free bottom node.
    \item Measure amplitude over time.
    \item To compare to the XPBD paper~\cite{10.1145/2994258.2994272} Fig.~2, I used the same parameters (just with reversed direction): rest length 1, compliance $\alpha = 10^{-3}$, rest position $x = -1$, initial position $x_0 = -0.5$, mass $m = 1$.
    \item Regardless of dt, both solvers converge in 1 iteration and follow the same curve.
    \item Analytic solution $x(t) = x_0\cos(\omega t)$, $\omega = \sqrt{k/m} = 31.62 \mathrm{s}^{-1}$ shown with dashed lines.
    \item Just as with the XPBD example, all damping is the side-effect of implicit discretization of the equations of motion.
\end{itemize}

\subsection{Iteration-count independence of stiffness}
\label{ssec:IterIndepStiff}
\begin{itemize}
    \item For each solver, sweep stiffness $k$ over several decades and measure how many iterations / substeps are needed to reach a fixed error threshold.
    \item In the compliance formulation, XPBD's per-iteration behaviour is roughly stiffness-independent (Fig.~6 in \cite{10.1145/2994258.2994272}).
    \item \autoref{fig:IterIndep} shows the settled shape of a pinned cloth at four iteration / substep counts for both solvers. For both methods the resting shape is essentially unchanged as the iteration count grows, so the converged configuration does not depend on how many iterations are spent.
    \item For both solvers the resting configuration is essentially unchanged across the sweep, indicating that the converged shape is independent of the iteration count.
\end{itemize}

\subsection{Ease of adding a new functionality}
\label{ssec:NewFunctionality}
I made Claude Code implement dihedral bending constraints in both solvers to compare the ease of adding a new functionality.
\begin{itemize}
    \item Both solvers have it implemented in their original papers - I follow this route
    \item Current bending constraint: a distance constraint as a skip-spring between two non-adjacent nodes. The task was to implement the new constraint and make it toggleable (bending as dihedral vs distance).
    \item Lines of code added:
        \begin{itemize}
            \item Shared: 70 lines
            \item Solver core: XPBD +39
            \item XPBD: 100 lines of code, no change to the core algorithm, just a new constraint type.
            \item VBD: 200 lines of code, including changes to the core algorithm to handle the new Hessian contribution.
        \end{itemize}
    \item whether a Hessian derivation was required (VBD: yes; XPBD: no, a gradient suffices),
    \item time to first correct simulation.
\end{itemize}
\begin{itemize}
    \item The metric is admittedly subjective but captures a real practitioner cost that the more quantitative experiments above do not.
\end{itemize}

\subsection{Coverage of different physical domains}
\label{ssec:DomainCoverage}
\begin{itemize}
    \item As PBD had already been around for almost 20, and PBD for 10 years, they also became industry standards for in-game physics. They have been implemented and used with many different domains (cloth, soft-bodies, rigid-bodies, fluids, hair) with a lot of different constraint types and forces. XPBD can handle a huge variety of parameters and value-imbalances, so it is a very flexible solver.
    \item VBD on the other hand is a very new method published in 2024. It is not yet widely used. In it's original paper it is mostly used for elastic body dynamics.
    \item I saw how it's not suitable for handling very hard constraints (infinite stiffness) - this issue is fixed in AVBD, introducing rigid-bodies to the method.
\end{itemize}
